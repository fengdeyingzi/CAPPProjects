## bmp图片转换器
**手机CAPP图形程序，解释执行模式**  
手机C默认采用此种模式运行程序，解释执行，免去编译的步骤，目前解释执行的C程序运行效率比较慢，如果要获取更高效率，可尝试创建CAPP编译程序（点击右上角菜单打开文件，在文件浏览器界面右下角创建工程，选择编译项目）。  
运行CAPP解释执行程序需要注意以下几点：  

* 必须实现init;event;pause;resume函数，此为系统的接口函数，这些函数的意义请参见api文档base.h
* volatile,register,auto,extern关键字可以使用，但实际会被忽略，goto语句仅支持向后goto
* 不支持函数指针
* printf打印的信息在c/print.txt文件里（可以直接点击编辑器右上角的控制台按钮查看）
* 不支持float和double类型
* #define只能做为表达式的一部分，其结果必须是一个值

**如何判断当前程序是解释执行还是编译运行？**  
在工程目录下的Android.mk是项目的配置文件，打开Android.mk文件，如果参数 RUN_MODE:=picoc 或者没有此参数， 则表示当前程序为解释执行程序，如果 RUN_MODE:=so 则表示编译执行，如果没有Android.mk文件，手机C将默认以标准C的方式运行。
